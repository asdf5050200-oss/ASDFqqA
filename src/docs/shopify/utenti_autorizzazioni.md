# 🔒 用户与权限 (Utenti e autorizzazioni) - 安全级别与团队控制指南

多租户 SaaS 架构的核心在于数据的严格隔离与内部操作权责的高精控管。本模块指导商户如何管理员工访问等级、定义业务职能组别以及确保数字主权安全。

---

## 1. 团队员工账号配置
- 为了避免员工交叉登陆及共用同一套 Root 凭证产生数据安全泄露，主账户可以自主添加多位辅助协作员工：
  - 支持配置每个员工的合法登陆用户名、官方联系方式、联系电邮及办公所在地物理标志。
- **添加上限**: Basic 套餐限制最多 5 位员工，Advanced 限制 15 位，Plus 套餐不限制。

## 2. 职能角色与精确垂直授权 (Ruoli e Permessi)
主账户可以将员工赋予相应的角色组别：
- **Admin (超级管理)**: 无限制访问所有配置、商品、财务及 API 设置权限。
- **Developer (技术开发)**: 允许管理域名对接、对接三方 App 接口插件及 Botble 联调套件，但默认无法修改对公收款银行账户 (Payout Accounts)。
- **Logistics (物流专员)**: 仅允许进入商品、订单、仓站 Locations 及配送模块。屏蔽敏感财务账单。
- **Marketing (市场营销)**: 仅允许创建促销活动、起草优惠卷 (Coupon Codes)、查看 GA4 事件追踪指标以及自定义默认邮件通知模块。

## 3. 安全合规与双因子认证 (2FA - Due fattori)
- **强监管模式**: 鉴于欧洲严格的 GDPR 用户私密数据保密以及金融欺诈防范规则，财务专员及管理员每次高敏操作（如：更新账单付款信用卡，发起大额订单退款，导出全量客户档案等）均需要两步验证或手机临时 OTP 签名。
- **登录轨迹审计 (Audit Log)**: 设置面板内可实时滚动展示当前租户下所有员工的物理登录行为回放（包括：访问时间、操作设备、IP归属大区以及当时操作的动作类型，防微杜渐）。

---

&nbsp;
> 🔐 **AI 安全总管 Stuart 智体提示**: 
> 任何在后台检测到的物理异地登录，其行为特征与 IP 轨迹都将无声经过 AI 智体的比对。如果登录行为出现极端跳跃（例如一小时内由巴黎跳至东南亚），系统可以自动锁止高危写入动作，直到店长双因子核销完毕后方可解锁，提供银行级的安全对局。
